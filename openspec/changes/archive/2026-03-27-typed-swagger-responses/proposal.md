## Why

The OpenAPI spec generated by NestJS Swagger lacks response schemas on most endpoints — list/paginated endpoints, CRUD responses, and analytics results all have untyped `@ApiResponse({ status: 200 })` decorators. This means `@hey-api/openapi-ts` generates `unknown`/`void` return types, making the generated client SDK effectively untyped for responses. Fixing this enables full end-to-end type safety between backend and frontend.

## What Changes

- Create response DTOs for every endpoint that currently returns untyped responses (transactions, transaction-categories, budgets, recurring-transactions, analytics, audit-log)
- Create a shared paginated response pattern with concrete DTOs for each list endpoint describing the `{ object, data, total, page, pageSize, hasMore }` envelope
- Create a shared `MessageResponseDto` for simple `{ message: string }` responses (delete operations, etc.)
- Extract inline/nested object shapes into dedicated DTO classes with `@ApiProperty` decorators
- Update all `@ApiResponse` decorators across all controllers to reference the correct response type
- No business logic changes — only Swagger decorator and DTO additions

## Capabilities

### New Capabilities

- `paginated-response-dtos`: Shared paginated response base and concrete paginated DTOs for each module's list endpoints
- `crud-response-dtos`: Response DTOs for single-resource CRUD endpoints (create, get-by-id, update) across transactions, transaction-categories, budgets, recurring-transactions, and audit-log modules
- `analytics-response-dtos`: Response DTOs for all transactions-analytics aggregation endpoints

### Modified Capabilities

- `swagger-response-coverage`: Extending coverage beyond auth/user to all remaining modules — every endpoint gets a typed response DTO

## Impact

- **Code**: New DTO files in `src/shared/dtos/` and each module's `dtos/` folder; updated `@ApiResponse` decorators in all 8 controllers
- **APIs**: No runtime behavior changes — only OpenAPI spec metadata improves
- **Dependencies**: No new dependencies
- **Systems**: Generated OpenAPI JSON will include full response schemas, enabling typed client generation with `@hey-api/openapi-ts`
